# Документация API бэкенда Lunex

В этом документе описаны все активные эндпоинты (маршруты) бэкенда, разделенные по модулям. Все защищенные клиентские маршруты требуют передачи заголовка `Authorization: Bearer <token>`, а маршруты админки требуют JWT-токен администратора.

## 1. Система и Общие (`/`)
* **GET `/`** — Проверка работоспособности (Health check), возвращает `{ status: 'ok' }`
* **GET `/api/status`** — Статистика системы, использование памяти, глобальные счетчики и состояние прокси. *(Защищено: требуется подпись/токен администратора)*

---

## 2. Авторизация (`/auth/*`)
Обработка процессов аутентификации через OAuth и Telegram.
* **GET `/auth/google`** — Инициирует вход через Google OAuth.
* **GET `/auth/google/callback`** — Обрабатывает редирект от Google OAuth и выдает токен.
* **POST `/auth/telegram`** — Валидация авторизации через Telegram Web App.
* **POST `/auth/verify-code`** — Проверка одноразового кода (например, для входа с ТВ/ПК).
* **GET `/auth/verify`** — Проверка валидности существующего JWT-токена.

---

## 3. Данные пользователя и Настройки (`/me/*`)
*Требуется авторизация пользователя (`authRequired`).*
* **GET `/me/`** — Получить профиль текущего пользователя.
* **GET `/me/avatar`** — Получить ссылку на аватарку пользователя.
* **GET / POST / DELETE `/me/likes`** — Управление лайкнутыми треками пользователя.
* **GET / POST / PUT / DELETE `/me/playlists`** — Управление кастомными плейлистами пользователя (CRUD).
* **POST / DELETE `/me/playlists/:id/tracks`** — Добавление или удаление треков из конкретного плейлиста.
* **GET / PUT `/me/settings`** — Управление настройками пользователя (например, тема интерфейса, язык).
* **GET / POST / DELETE `/me/search-history`** — Получить, добавить или очистить историю поиска.
* **GET / POST / DELETE `/me/listening-history`** — Получить, добавить или очистить историю прослушиваний.
* **GET `/me/badges`** — Получить заработанные значки (ачивки) пользователя.

---

## 4. Интеграция YouTube Music (`/api/yt/*`)
*Кэшируемые эндпоинты для получения данных из YouTube Music.*
* **GET `/api/yt/search`** — Поиск треков.
* **GET `/api/yt/search-artists`** — Поиск исполнителей (артистов).
* **GET `/api/yt/search-playlists`** — Поиск плейлистов в YT.
* **GET `/api/yt/artist`** — Получить базовую информацию об артисте.
* **GET `/api/yt/artist-full`** — Получить полный профиль артиста (альбомы, синглы, похожие артисты).
* **GET `/api/yt/artist-songs`** — Получить все песни конкретного артиста.
* **GET `/api/yt/playlist`** — Получить метаданные плейлиста.
* **GET `/api/yt/playlist-tracks`** — Получить треки из плейлиста.
* **GET `/api/yt/album`** — Получить метаданные альбома.
* **GET `/api/yt/album-tracks`** — Получить список треков в альбоме.
* **GET `/api/yt/upnext`** — Получить рекомендации "Up Next" / Радио из YouTube Music.

---

## 5. Интеграция SoundCloud (`/api/sc/*`)
*Кэшируемые эндпоинты для получения данных из SoundCloud.*
* **GET `/api/sc/search`** — Поиск треков.
* **GET `/api/sc/search/users`** — Поиск пользователей/исполнителей в SC.
* **GET `/api/sc/search/playlists`** — Поиск плейлистов в SC.
* **GET `/api/sc/stream`** — Получить прямую ссылку на MP3-стрим трека.
* **GET `/api/sc/user`** — Получить профиль пользователя по ссылке (URL).
* **GET `/api/sc/user-by-id`** — Получить профиль пользователя по его ID.
* **GET `/api/sc/user-by-id/tracks`** — Получить треки, загруженные пользователем.
* **GET `/api/sc/user-by-id/playlists`** — Получить плейлисты, созданные пользователем.
* **GET `/api/sc/playlist-tracks`** — Получить треки из SC плейлиста.
* **GET `/api/sc/likes`** — Получить лайкнутые треки SC пользователя.
* **GET `/api/sc/playlists`** — Получить все плейлисты SC пользователя.
* **GET `/api/sc/related`** — Получить похожие треки (Радио SC).
* **GET `/api/sc/artist-tracks`** — Алиас (копия) для user-by-id/tracks.

---

## 6. Темы оформления (`/themes/*`)
* **GET `/themes/`** — Получить список всех доступных пользовательских тем.
* **POST `/themes/`** — Загрузить/опубликовать новую кастомную тему. *(Требуется авторизация)*
* **POST `/themes/:id/download`** — Увеличить счетчик скачиваний для конкретной темы.

---

## 7. Админ Панель (`/api/admin/*`)
*Требует JWT-токен администратора. Используется исключительно в веб-панели управления.*
* **GET `/api/admin/users/recent`** — Получить список из 50 последних активных пользователей.
* **GET `/api/admin/users/:id/details`** — Deep-dive: получить профиль юзера, его последние 50 поисков и 50 прослушанных треков.
* **POST / DELETE `/api/admin/users/:id/ban`** — Забанить или разбанить пользователя.
* **GET `/api/admin/proxies`** — Получить живое состояние пула прокси (IP-адреса, кулдауны, счетчики ошибок).
* **POST `/api/admin/proxies`** — Добавить новую прокси "на горячую" в пул.
* **DELETE `/api/admin/proxies`** — Удалить прокси из пула.
* **POST `/api/admin/proxies/reset`** — Сбросить таймеры кулдауна и счетчики ошибок для всех прокси-серверов.
* **GET `/api/admin/insights/top-searches`** — Получить топ-50 глобальных поисковых запросов.
* **GET `/api/admin/insights/top-tracks`** — Получить топ-50 самых популярных треков среди всех юзеров.
* **GET `/api/admin/logs`** — Получить последние 200 строк консольных логов (перехватываются динамически).
* **GET `/api/admin/metrics/history`** — Получить последние 50 тиков статистики (RAM / Активные юзеры) для графиков в дашборде.
* **DELETE `/api/admin/cache`** — Жестко очистить весь in-memory node-кэш приложения.
* **POST `/api/admin/restart`** — Отправить сигнал `process.exit(0)` для мягкой перезагрузки через PM2/Nodemon.
